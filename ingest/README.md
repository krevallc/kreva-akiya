# エンリッチ層 PoC（KREVA 空き家検索）

住所を入れると、**緯度経度・市街化調整区域/用途地域・地価・災害ハザード**を取得し、
**ハザードを重ねた地図HTML**を生成する Python PoC。空き家ページ最大の差別化ポイントを、
物件データ源に依存せず先行検証するためのもの。

## セットアップ
```bash
cd ingest
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env      # REINFOLIB_API_KEY を設定（下記）
```

## APIキー（無料）
規制/地価/災害の取得には不動産情報ライブラリのAPIキーが必要。
1. https://www.reinfolib.mlit.go.jp/api/request/ で申請（無料）
2. 発行キーを `.env` の `REINFOLIB_API_KEY=` に設定

> キー未設定でも「ジオコーディング＋地図＋ハザードタイル重畳」は動作する（検証済み）。

## 実行
```bash
python poc.py "岡山県総社市中央一丁目1-1"
python poc.py "岡山県井原市七日市町..." --zoom 15 --json
```
- 標準出力に区域区分/用途地域/建蔽率・容積率/最寄り地価/災害該当を表示。
- `out/map_*.html` を生成。ブラウザで開き、右上のレイヤ切替でハザード重畳を確認。

## 構成
| ファイル | 役割 |
|---|---|
| `geocode.py` | 国土地理院 住所検索API（キー不要）で住所→緯度経度 |
| `tiles.py` | 緯度経度↔XYZタイル変換・点内包判定（依存なし） |
| `reinfolib.py` | 不動産情報ライブラリAPIクライアント（区域区分/用途地域/地価/災害） |
| `poc.py` | 入口。ジオコーディング→エンリッチ→地図HTML生成 |

## 使用データ・API（すべて商用可）
- 国土地理院 住所検索API / 地理院タイル（淡色・標準）
- 国土交通省 不動産情報ライブラリ API
  - XKT001 区域区分（市街化区域/市街化調整区域）
  - XKT002 用途地域（建蔽率・容積率）
  - XPT002 地価公示・地価調査ポイント
  - XKT026/027/028/029 洪水/高潮/津波/土砂
- 国土地理院 重ねるハザードマップ ラスタタイル（洪水/土石流/急傾斜/津波/高潮）

## 注意（スクレイピングとは無関係の合法データのみ）
このPoCは政府提供の公開API/タイルのみを使用。物件データのスクレイピングは別モジュール（今後 `scrapers/`）で、
robots/規約順守・低頻度・出典明記のガードレール下で実装する。
