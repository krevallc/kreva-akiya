# デプロイ手順（空き家検索機能）

> 対象：kreva.co.jp（mixhost・WordPress）＋ GitHub Actions（取り込みの定期実行）
> 前提：mixhost への kreva.co.jp 移行が完了していること（移行手順書は別紙）。

---

## 全体像

```
GitHub（このフォルダをプライベートrepo化）
  └─ Actions: 週1回（月曜03:00 JST）
       scrapers収集 → APIエンリッチ → kreva.co.jp/wp-json へ upsert
                                           │
mixhost WordPress ←──────────────────────────┘
  └─ kreva-akiya プラグイン（CPT akiya・/akiya/ 検索・詳細・地図）
```

---

## STEP 1：WordPress 側（mixhost移行後に1回だけ）

1. **プラグイン設置**：`wordpress/kreva-akiya/` フォルダごと
   `wp-content/plugins/` へアップロード（FTP または cPanelファイルマネージャ）。
2. 管理画面 → プラグイン → **「KREVA 空き家検索」を有効化**。
3. 設定 → パーマリンク → そのまま**保存**（リライト反映）。
4. `https://kreva.co.jp/akiya/` を開き、地図が表示されればOK。
5. **アプリケーションパスワード発行**（取り込み用）：
   - ユーザー → プロフィール → 下部「アプリケーションパスワード」
   - 名前例 `akiya-ingest` → 発行された24桁を控える（**この画面でしか見られない**）。

## STEP 2：GitHub リポジトリ作成（1回だけ）

このフォルダ（`空き家検索機能/`）をリポジトリのルートとして push する。

```bash
cd "~/ObsidianVault/Projects/KREVA LLC/Website/KREVA JP Website改修/空き家検索機能"
git init
git add .
git commit -m "KREVA 空き家検索 初期コミット"
# GitHubでプライベートリポジトリ（例: kreva-akiya）を作成してから：
git remote add origin git@github.com:<アカウント>/kreva-akiya.git
git branch -M main
git push -u origin main
```

> `.gitignore` 済み：`ingest/.env`（APIキー）と `ingest/out/` はコミットされない。

## STEP 3：GitHub Secrets 登録（1回だけ）

リポジトリ → Settings → Secrets and variables → **Actions** → New repository secret：

| Name | 値 |
|---|---|
| `REINFOLIB_API_KEY` | 不動産情報ライブラリのAPIキー（`.env` と同じ値） |
| `WP_BASE_URL` | `https://kreva.co.jp` |
| `WP_USER` | WordPressのユーザー名 |
| `WP_APP_PASSWORD` | STEP1-5 のアプリケーションパスワード |

## STEP 4：初回実行（手動・少量テスト → 本投入）

1. リポジトリ → **Actions** → `akiya-ingest` → **Run workflow**
2. まずテスト：`source=ok_smile` / `cities=33208` / `push=true` / `limit=2`
   → 成功したら WordPress 管理画面「空き家物件」に総社2件が入っているのを確認。
3. 本投入：`source=all` / `cities=all` / `push=true` / `limit`空欄
   → 5市＋県内参加自治体の全物件が入る（100件超・1時間程度）。
4. `https://kreva.co.jp/akiya/` で地図にピンが並ぶことを確認。

## STEP 5：以後の運用（自動）

- **毎週月曜 03:00 JST に自動実行**（新規物件の追加・既存の更新。source_id で冪等upsert）。
- 実行ログ：GitHub Actions の各Run。収集JSONはArtifactsに30日保存（デバッグ用）。
- 失敗時はGitHubから通知メールが来る（Settings → Notifications 確認）。

## ローカルから手動実行したい場合（GitHub不要の代替）

```bash
cd ingest
source .venv/bin/activate   # 初回: python3 -m venv .venv && pip install -r requirements.txt
# .env に REINFOLIB_API_KEY / WP_BASE_URL / WP_USER / WP_APP_PASSWORD を設定済みであること
python run_ingest.py --source all --cities all --enrich --push
```

---

## 運用メモ・注意

- **頻度は週1のまま増やさない**（対象サイトへの負荷配慮＝ガードレール）。
- 高梁の「交渉状況」が成約系になった物件は status=成約済 で upsert される（非表示にしたい場合はWP側でstatusタクソノミーを使って絞る）。
- 掲載が消えた物件の自動アーカイブは未実装（Phase2候補：source_id が今回分に無い物件を `成約/掲載終了` へ落とす処理）。
- APIキーを漏らした場合は不動産情報ライブラリで再発行し、GitHub Secret と `.env` を更新。
- プラグイン更新時は `wordpress/kreva-akiya/` を再アップロード（上書き）するだけ。
