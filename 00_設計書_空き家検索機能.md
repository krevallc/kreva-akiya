## 1. データ源の整理（最重要）

### 1-A. 物件リスト（＝唯一の難所）
| 優先 | ソース | 内容 | 方針 |
|---|---|---|---|
| ★主軸 | **自治体オープンデータ / 国交省 Project LINKS・空き家バンク標準フォーマット** | 参加自治体がCSV/APIで公開する空き家バンクデータ | 標準フォーマットに正規化して取り込み。合法・保守が軽い |
| ○補完 | **個別自治体サイトのスクレイピング** | オープンデータ非対応の自治体バンクHTML | ソース別アダプタで個別対応。保守が重いので必要分だけ |
| △誘導 | **全国版空き家バンク（LIFULL / アットホーム）** | 国交省指定の2大集約（約1,030自治体・58%） | **機械取得はToS違反→データ流用しない。検索結果から"外部詳細"としてリンク誘導のみ** |
| ★自社 | **KREVA自社物件** | 岡山等で購入・リノベ・民泊・転売する物件 | 手入力（CPTに直接登録）。最優先で目立たせる |

> ※「全国網羅」は Project LINKS 等オープンデータの整備進捗に依存。だからこそ段階導入が現実的。

### 1-B. 周辺情報エンリッチ（＝自作で高品質に作れる差別化ポイント）
| 情報 | ソース（すべて無料・商用可） | 取得方法 |
|---|---|---|
| 市街化区域/**市街化調整区域**・用途地域・防火地域・地区計画 | **不動産情報ライブラリAPI**（国交省） | 緯度経度→区域区分/用途地域 GeoJSON |
| **災害リスク**（洪水・土砂・津波・高潮）地図オーバーレイ | **重ねるハザードマップ タイル**（disaportal.gsi.go.jp／地理院タイル仕様） | Leafletにタイルレイヤ重畳＋トグル |
| 災害危険区域・指定緊急避難場所 | 不動産情報ライブラリAPI | 緯度経度周辺を取得 |
| **地価**（地価公示・都道府県地価調査）・周辺取引価格 | 不動産情報ライブラリAPI | 近傍ポイント取得→価格の妥当性表示 |
| 人口推計（500mメッシュ）・駅乗降客数 | 不動産情報ライブラリAPI | 参考指標として表示 |
| 住所→緯度経度（ジオコーディング） | **国土地理院ジオコーディングAPI**（無料） | 取り込み時に付与 |
| ベース地図タイル | **地理院タイル**（無料・商用可） | Leaflet |

> APIキー：不動産情報ライブラリは利用申請が必要（無料）。→ Phase1 の最初のタスクで申請。

## 2. アーキテクチャ（Phase1：WordPressネイティブ）

```
[取り込みバッチ]  ──（正規化・ジオコーディング・API付加）──▶  [WordPress DB]  ──▶  [フロント]
 自治体OD / API                                              CPT: akiya            検索地図＋詳細
 スクレイパ（必要分）                                          + meta/taxonomy       Leaflet+ハザード
 不動産情報ライブラリAPI                                                            自社物件導線
```

- **取り込み**：まずはWP-Cron＋PHPで可。全国スケール時は外部cron/VPS（Python）→WP REST APIで投入に移行（shared hostingのWP-Cronは非力なため）。
- **保存**：カスタム投稿タイプ（CPT）`akiya`。数万件までならWPで十分（メタをインデックス）。
- **フロント**：テーマ内テンプレート＋Leaflet。プラグイン過多を避け軽量に。

### 2-1. データモデル（CPT `akiya`）
- **taxonomy**：`都道府県` / `市区町村` / `物件種別`（戸建/土地/…）/ `ステータス`（自社/バンク/成約済）
- **meta（主なもの）**：
  - 基本：`価格` `土地面積` `建物面積` `築年` `間取り` `構造` `所在地(表示用)` `緯度` `経度`
  - 権利/規制：`市街化区域区分`（区域内/調整区域）`用途地域` `建ぺい率` `容積率`
  - 災害：`洪水想定` `土砂災害警戒` `津波` `高潮`（フラグ＋想定浸水深など）
  - 参考：`近傍地価` `周辺取引価格レンジ`
  - 出所：`ソース名` `ソースURL` `ソース物件ID` `最終確認日`
  - 自社：`is_kreva`（自社物件フラグ）`kreva用途`（リノベ/民泊/転売）`相談ステータス`

### 2-2. 取り込みパイプライン（1件の流れ）
1. 収集（OD API or スクレイパ）→ 標準フォーマットへ正規化
2. 重複判定（`ソース名＋ソース物件ID`でupsert）
3. ジオコーディング（住所→緯度経度）
4. 不動産情報ライブラリAPIで 区域区分/用途地域/地価/ハザード を付加
5. CPTへupsert、`最終確認日`更新（消えた物件は`成約/掲載終了`へ）

## 3. フロントエンド

### 3-1. 検索・一覧ページ
- 地図（Leaflet＋地理院タイル）に物件ピン。**ハザードオーバーレイのトグル**（洪水/土砂/津波）。
- フィルタ：エリア（県→市）/ 価格 / 面積 / 種別 / **市街化調整区域の有無** / 災害リスク低 / 自社物件のみ。
- カード一覧（地図と連動ハイライト）。**自社物件は上位固定＋バッジ**。

### 3-2. 詳細ページ（single-akiya）
- 物件スペック表、写真、地図。
- **規制**：市街化区域/調整区域・用途地域・建ぺい/容積。
- **災害**：洪水/土砂/津波の想定＋ハザード地図、最寄り避難場所。
- **相場**：近傍地価・周辺取引価格で価格の妥当性。
- 出所リンク（自治体バンク元ページ）＋`最終確認日`。
- **CTA**：「この物件についてKREVAに相談」フォーム（info@kreva.co.jp）。自社物件は購入/民泊/リノベ導線。

### 3-3. 自社導線（ハイブリッドの肝）
- 検索で集客 → 詳細/一覧に**KREVAの実績（リノベ前後・民泊運営）**と相談CTAを常設。
- 自社物件は「KREVAセレクト」等で別枠強調。

## 4. 法務・運用の注意
- **スクレイピングは対象サイトの規約・robots・負荷を確認**。オープンデータを最優先し、スクレイピングは規約が許す自治体のみ・低頻度で。
- **LIFULL/アットホーム全国版バンクのデータは流用しない**（ToS）。リンク誘導のみ。
- 各データの**出典表記**（国交省・国土地理院・各自治体）をページに明記。
- 物件情報は**必ず出所と最終確認日を表示**し「最新は元サイト/自治体で要確認」の注記。誤情報リスク回避。
- 個人情報・所有者情報は扱わない（公開情報のみ）。

## 8. Phase1 データ棚卸し結果（2026-07-02 Web調査）

### 結論：岡山では「オープンデータ」は実質なし → 取り込みはHTMLスクレイピング中心へ方針転換
- おかやまオープンデータカタログ(data eye)に「住宅・空き家」カテゴリはあるが、**5市の空き家バンク物件データはオープンデータ化されていない**（西粟倉村のPDF程度）。
- 各市の実データは、県レベル集約サイト or 各市サイト or 全国版バンク基盤にHTML掲載されている。

### 各市の実データの在り処と推奨取り込み経路
| 市 | 実データの在り処 | 推奨取り込み | 注意 |
|---|---|---|---|
| 岡山市 | **住まいる岡山**(ok-smile.jp) ※市公式は案内のみ | 住まいる岡山をスクレイプ | 規約グレー |
| 倉敷市 | 住まいる岡山 ＋ 市公式バンク登録物件一覧 | 住まいる岡山 or 市公式HTML | 規約グレー |
| 総社市 | 住まいる岡山 ＋ at-home自前サイト | **住まいる岡山**（at-homeは回避） | at-home=全国版ToS× |
| 井原市 | **井原Life**(ibaragurashi.jp/house) ＋ at-home | 井原Life直接スクレイプ | 詳細URLがハッシュ・写真なし・ページング要 |
| 高梁市 | LIFULL ＋ 市委託(梁クラス)、住まいる岡山リストに無し | **要追加調査**（市公式/委託ルート） | 別ルート確定が必要 |

### 重要な発見
- **「住まいる岡山」を1本攻略すれば 岡山市・倉敷市・総社市 の3市を一挙にカバー可能**（県宅建協会＋不動産協会運営・18自治体集約。総社/倉敷/岡山市を含む）。→ Phase1の主戦場。
- **全国版バンク基盤（at-home / LIFULL）はデータ流用不可（ToS）** → 総社/井原/高梁の一部はこれを避け、市公式系ルートを使う。
- 規約：住まいる岡山・井原Life とも**スクレイピング明示禁止は確認できず＝グレー**（免責/プライバシー精読が必要）。

### 姿勢の選択肢（要判断）
- **(A) 直接交渉先行**：住まいる岡山運営（岡山県宅建協会）・各市へ「空き家データ連携」を打診。合法・安定。KREVAは地元で実際に空き家を買う事業者＝交渉で有利。時間はかかる。
- **(B) スクレイピング先行**：明示禁止のない範囲で低頻度・出典明記で取得。早いがグレー・保守リスク。
- **(C) 折衷（推奨）**：まず**自社物件＋周辺情報エンリッチ**でページを立ち上げ、外部物件は当面**リンク誘導**。並行して(A)交渉と(B)PoCを進め、許諾/目処が立った経路から実データ取り込みを拡張。

### 確実に作れる部分（データ在り処に依存しない）
- 周辺情報エンリッチ（**市街化調整区域・用途地域・災害ハザード・地価**）は 不動産情報ライブラリAPI＋重ねるハザードマップ＋地理院ジオコーディング で**完全に高品質に自作可能**。→ ここがKREVAページ最大の差別化。物件データ問題と切り離して先行実装できる。
